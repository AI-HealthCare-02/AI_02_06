## Phase 0: 환경 준비

### 0-1. .env 파일 설정

프로젝트는 심볼릭 링크를 통해 `envs/` 폴더의 환경 파일을 사용합니다.

```bash
cd E:\Project\Team_Project\OZ-Final\AH_02_06

# 개발 환경 (Docker Compose)
# .env -> envs/.local.env (심볼릭 링크)
```

**필수 입력 항목:**

| 변수 | 설명 | 예시 |
|------|------|------|
| `ENV` | 실행 환경 | `local`, `dev`, `prod` |
| `SECRET_KEY` | JWT 비밀키 (32자 이상) | `python -c "import secrets; print(secrets.token_hex(32))"` |
| `DB_PASSWORD` | PostgreSQL 비밀번호 | (팀 공유) |
| `KAKAO_CLIENT_ID` | 카카오 앱 REST API 키 | Kakao Developers 콘솔 |
| `KAKAO_CLIENT_SECRET` | 카카오 앱 Secret | Kakao Developers 콘솔 |

**환경별 URL 설정:**

| 변수 | local | dev (Docker) | prod |
|------|-------|--------------|------|
| `FRONTEND_URL` | `http://localhost:3000` | `http://localhost` | `https://도메인` |
| `API_BASE_URL` | `http://localhost:8000` | `http://fastapi:8000` | `http://fastapi:8000` |
| `KAKAO_REDIRECT_URI` | `http://localhost:3000/auth/kakao/callback` | `http://localhost/auth/kakao/callback` | `https://도메인/auth/kakao/callback` |
| `NEXT_PUBLIC_API_BASE_URL` | `http://localhost:8000` | (빈 문자열) | (빈 문자열) |
| `NEXT_PUBLIC_ENV` | `local` | `dev` | `prod` |

**카카오 디벨로퍼 설정:**

Redirect URI를 카카오 디벨로퍼 콘솔에 등록해야 합니다:
- 개발: `http://localhost/auth/kakao/callback`
- 프로덕션: `https://도메인/auth/kakao/callback`

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 카카오 로그인 실패 (dev/prod) | KAKAO_CLIENT_ID 미설정 또는 Redirect URI 미등록 | .env 파일 확인, 카카오 디벨로퍼 콘솔에 Redirect URI 등록 |
| 개발자 로그인 버튼 안보임 | `NEXT_PUBLIC_ENV=prod` 설정됨 | dev 환경에서는 `NEXT_PUBLIC_ENV=dev`로 설정 |
| ECONNREFUSED 에러 | Docker 네트워크 설정 오류 | `API_BASE_URL=http://fastapi:8000` 확인 |
| DB 연결 오류 | DB_PASSWORD 오류 또는 네트워크 | 52.78.62.12 접근 가능 확인 |
| 502 Bad Gateway | FastAPI 아직 시작 안됨 | `docker-compose logs fastapi` 확인 |
| OCR 타임아웃 | CLOVA API 키 미설정 | .env에 `CLOVA_OCR_SECRET_KEY`, `CLOVA_OCR_INVOKE_URL` 확인 |
| AI 응답 없음 | OpenAI API 키 미설정 | .env에 `OPENAI_API_KEY` 확인 |
| Mock 카카오 안됨 (dev 환경) | ENV=dev에서는 실제 카카오 사용 | Mock 테스트는 `ENV=local`에서 수행 |

## 변경 사항별 Docker 반영 방법

### 변경 유형별 필요 작업

| 변경 유형 | 필요 작업 | 명령어 |
|----------|----------|--------|
| 소스 코드 변경 | 없음 (자동 반영) | - |
| `.env` 환경변수 변경 | 컨테이너 재생성 | `docker-compose up -d` |

### 1. 소스 코드 변경 시 (재시작 불필요)

개발 환경에서는 **Hot Reload**가 설정되어 있어 코드 변경 시 자동 반영됩니다.

| 서비스 | Hot Reload 방식 | 볼륨 마운트 |
|--------|----------------|-------------|
| FastAPI | `uvicorn --reload` | `./app:/app/app` |
| Next.js | HMR (Hot Module Replacement) | `./medication-frontend:/app` |

```bash
# 코드 수정 후 저장하면 자동 반영
# 별도 명령어 불필요
```

---

### 2. .env 환경변수 변경 시

환경변수는 **컨테이너 생성 시점**에 로드되므로 컨테이너 재생성이 필요합니다.

```bash
# 권장: 변경 감지하여 필요한 서비스만 재생성
docker-compose up -d

# 특정 서비스만 재생성
docker-compose up -d fastapi
docker-compose up -d frontend
docker-compose up -d ai-worker
```

**주의**: `docker-compose restart`는 환경변수를 반영하지 않습니다!

| 명령어 | .env 반영 | 설명 |
|--------|----------|------|
| `docker-compose restart` | X | 프로세스만 재시작 |
| `docker-compose up -d` | O | 컨테이너 재생성 |

**서비스별 환경변수 매핑:**

| 환경변수 | 사용 서비스 |
|----------|-----------|
| `ENV`, `SECRET_KEY`, `DB_*`, `KAKAO_*` | fastapi |
| `NEXT_PUBLIC_*` | frontend |
| `OPENAI_API_KEY`, `CLOVA_*` | ai-worker |

### 4. Dockerfile 또는 의존성 변경 시

`Dockerfile`, `requirements.txt`, `package.json` 등 변경 시 이미지 재빌드가 필요합니다.

```bash
# 변경된 서비스만 재빌드
docker-compose up -d --build

# 특정 서비스만 재빌드
docker-compose up -d --build fastapi
docker-compose up -d --build frontend

# 전체 재빌드 (클린)
docker-compose down
docker-compose up -d --build
```

---

### 5. 완전 초기화 (문제 발생 시)

캐시나 볼륨 문제로 정상 동작하지 않을 때:

```bash
# 컨테이너 + 볼륨 삭제 후 재빌드
docker-compose down -v
docker-compose up -d --build
```
